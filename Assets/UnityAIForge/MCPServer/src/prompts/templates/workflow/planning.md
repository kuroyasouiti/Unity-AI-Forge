# 企画 ワークフローガイド (Unity-AI-Forge v{VERSION})

ゲームアイディアを構造化された企画書にまとめ、後続フェーズへの明確な入力を作るためのガイドです。

---

## 概要

企画フェーズの目的は「何を作るか」を明確にすることです。曖昧なアイデアを構造化し、コアループ・メカニクス・シーン構成・マイルストーンを定義することで、以降の設計・実装フェーズがブレなく進みます。

**企画フェーズの原則:**
- ゲームのコアループ（プレイヤーが繰り返す基本行動）を最初に定義する
- メカニクスはコアループを支える要素として整理する
- シーン一覧とフォルダ構成はこの段階で決める
- マイルストーンを定義し、検証ポイントを明確にする
- MCPツールの使用は最小限（ドキュメント保存程度）

---

## パイプライン位置

```
[企画] → 設計 → プロジェクト初期設定 → シーン構築 → プロトタイプ → アルファ → ベータ → リリース
```

## 企画書テンプレート

以下の項目を順番に埋めていくことで、企画書が完成します。

### 1. 基本情報

| 項目 | 内容 |
|------|------|
| タイトル（仮） | |
| ジャンル | platformer_2d / rpg_turnbased / puzzle / shooter_2d / action_2d / card_game 等 |
| ターゲットプラットフォーム | PC / Mobile / Console |
| 開発規模 | 個人 / 小規模チーム (2-5人) / 中規模 (5-15人) |
| 想定プレイ時間 | |

### 2. コンセプト

```
一言コンセプト: （20文字以内でゲームの本質を表現）
詳細説明: （3-5文で世界観・プレイ感を説明）
参考作品: （類似ゲーム・インスピレーション元を列挙）
```

### 3. コアループ

ゲームの核となる「プレイヤーが繰り返す行動サイクル」を定義します。

```
[入力] → [行動] → [結果] → [報酬/フィードバック] → (繰り返し)
```

**例 (2Dプラットフォーマー):**
```
移動・ジャンプ → 敵を踏む/障害物を避ける → ゴールに到達 → スコア・次ステージ解放
```

**例 (ターン制RPG):**
```
探索 → エンカウント → コマンド選択・戦闘 → 経験値・アイテム獲得 → 成長・新エリア解放
```

### 4. メカニクス一覧

コアループを支えるゲームメカニクスをリストアップします。

| メカニクス | 優先度 | 説明 | 関連ガイド |
|-----------|--------|------|-----------|
| 状態管理 | 必須 | プレイヤー/敵の状態遷移 | `game_mechanics_guide(mechanic='state_machine')` |
| インベントリ | 高 | アイテム管理 | `game_mechanics_guide(mechanic='inventory')` |
| セーブ/ロード | 中 | 進行保存 | `game_mechanics_guide(mechanic='save_load')` |
| 敵AI | 高 | 敵の行動ロジック | `game_mechanics_guide(mechanic='enemy_ai')` |
| UI/UX | 必須 | メニュー・HUD | `game_mechanics_guide(mechanic='ui_ux')` |
| レベルデザイン | 高 | ステージ設計 | `game_mechanics_guide(mechanic='level_design')` |

### 5. シーン構成

ゲームに必要なシーンをリストアップします。

```
Assets/Scenes/
  Boot.unity           # 起動・初期化
  UI/
    MainMenu.unity     # タイトル画面
    GameHUD.unity      # ゲーム中HUD
    PauseMenu.unity    # ポーズ画面
    GameOver.unity     # ゲームオーバー
  Levels/
    Level01.unity      # ステージ1
    Level02.unity      # ステージ2
  Managers.unity       # 永続マネージャー
```

詳細なマルチシーン設計は `game_workflow_guide(phase='scene_structure')` を参照してください。

### 6. マイルストーン定義

| マイルストーン | 目標 | 検証基準 |
|-------------|------|---------|
| プロトタイプ | コアループが動く | プリミティブで基本操作→ゴール到達が確認できる |
| アルファ | ゲームロジック完成 | SO・状態管理・イベント接続が動作する |
| ベータ | 演出・品質完成 | 実アセット・VFX・Audio・フィードバックが統合済み |
| リリース | 出荷可能 | 全チェック通過・ビルド成功 |

## MCPツール使用例

企画フェーズではMCPツールの使用は最小限ですが、ドキュメントの保存に使えます。

```python
# 企画書をプロジェクト内に保存
unity_asset_crud(operation='create',
    assetPath='Assets/Documents/GameDesignDoc.md',
    content='# ゲーム企画書\n\n## コンセプト\n...')

# フォルダ構造のスケルトンを先行作成
unity_asset_crud(operation='create',
    assetPath='Assets/Documents/TechSpec.md',
    content='# 技術仕様書\n\n## アーキテクチャ\n...')
```

## チェックリスト

- [ ] タイトル（仮）を決めた

- [ ] 各メカニクスに優先度（必須/高/中/低）を付けた

- [ ] シーン一覧を作成した
- [ ] フォルダ構成案を作成した

### マイルストーン
- [ ] プロトタイプ/アルファ/ベータ/リリースの目標を定義した
- [ ] 各マイルストーンの検証基準を明記した

## 次のフェーズへ

企画書が完成したら、以下の順序で進みます:

1. **設計フェーズ** (`game_workflow_guide(phase='design')`) - デザインパターン選定・UML設計
2. **プロジェクト初期設定** (`game_workflow_guide(phase='project_setup')`) - タグ・レイヤー・フォルダ構造
3. **シーン構築** (`game_workflow_guide(phase='scene_building')`) - シーン階層・Manager・Canvas 骨格の構築
4. **プロトタイプ** (`game_workflow_guide(phase='prototype')`) - コアループの動作検証
